Introduction

This project aims to automate the point scoring of physical King Domino boards. From a raw overhead image of an assembled board to the corresponding total point score of that board.

Link to code repository: https://github.com/021021-dani/Duas_mini_projekt

First, the board image is segmented into a 5x5 grid of individual terrain tiles.
For each tile, numerical features are extracted in the form of a 20-value Hue-Saturation-Value (HSV) color histogram.
A Support Vector Machine (SVM) categorizes each tile into one of the 8 predefined terrain classes including "Empty"-class.
Crown symbols are detected and counted on each individual tile. The crown symbols are important since they act as score multipliers.
To identify connected regions of the same terrain type, a Breadth-First Search (BFS) algorithm traverses the 5x5 board grid.
Once both terrain-class and crown count for each tile has been determined, and connected regions has been mapped, the final score is calculated in accordance with the game’s rules.

Preprocessing and data acquisition

To prevent data leakage, grouped dataset splitting strategy is used. Data is split at the board level rather than the tile level. This ensures that visually correlated paired images, such as the same board photographed with and without physical 3D objects, are placed in the same split. A dedicated hold-out test set consisting of 12 boards is entirely excluded from the training phases.

The dataset is prepared by organizing and processing the board images. The images below consist of two parts: (left image below) examples of the created board folders, showing the folder structure containing multiple boards, and (right image below) a tile extraction example from a single board, illustrating how each board is split into individual tiles and stored separately. This step prepares the data for feature extraction and model training.


  
Feature extraction
After segmentation for each tile image, a feature vector is extracted using colour histogram in the HSV colour space. The histogram consists of 10 bins for Hue, 5 for Saturation, and 5 for Value, resulting in a 20-dimensional feature vector. Terrain classification 
The extracted feature vectors are used to train a classifier for terrain classification. A support vector Machine (SVM) with a Radial Basis Function (RBF) kernel is applied to classify each tile into one of terrain classes. The RBF kernel is defined as:
K (x_i , x_j ) e^(-γ(∥ x_i- x_j∥) )

where x_i and x_j are feature vectors,  ∥x_i-x_j ∥^2 is the squared Euclidean distance, and γ  controls how quickly similarity decreases. This kernel measures the similarity between samples and enables non-linear separation of the data. The SVM model is trained using a grouped k-fold cross-validation strategy to prevent data leakage and is evaluated based on mean classification accuracy.

Feature Space Visualization using PCA
To better understand the distribution of the extracted feature vectors, Principal Component Analysis (PCA) is applied to reduce the high-dimensional feature space to 2D and 3D. This allows visual inspection of how different terrain classes are grouped and separated.

     
               Figure 7: 3D PCA Visualization of Terrain Features   
As shown in figure 7, terrain classes such as water, field, and grass form relatively distinct clusters, while classes like mine, home, and swamp show some overlap. Although overlap is present due to dimensionality reduction, the features still contain sufficient information for classification, which is reflected in the high SVM performance. The overlap observed in PCA projections is expected, as dimensionality reduction may lost information compared to the original feature space.

Crown detection
To determine the score multipliers for each region, crown symbols are counted on individual terrain tiles using template matching and Non-Maximum Suppression (NMS).
Template matching process:
 
First, each tile image is converted from BlueGreenRed (BGR) to the HueSaturationValue (HSV) color space. Both the BGR image (figure x*) and the HSV image (figure x*) are then visualized side by side. The HSV representation improved the manual detection of crowns, since they were highlighted in red.
In the image below, the bottom crown was chosen and cut out to become a template, therefore the correlationScore is 100% when finding template matches:
 
Afterward, the collection chosen reference crown templates are evaluated one after another across the tile image using normalized cross-correlation. 
The image below shows template matching with a visualization of best matches for a given template with a correlation heatmap:
 
When a template's correlation score exceeds a designated threshold, a crown bounding box is drawn.  

Non maximum suppression 
Because a single crown might trigger multiple overlapping matches, NMS is applied. NMS filters out duplicate detections by discarding bounding boxes that share an overlap greater than a defined limit, retaining only the strongest match.
The image below is an illustration of multiple bounding boxes on a tile before and after applying NMS:

Parameter optimization
The correlation score threshold was originally set to 0.60, and was hereafter iteratively moved up after the results on the training set had been analyzed. The correlation score threshold was set based on balancing the fewest false positives, while requiring more templates if moved higher. 
The image below contains the false positive with the highest correlation score of 0.70 and 0.72 found on forest tile 4,0 from board 38:
 
To avoid false positives, the correlation score threshold was finally set to a minimum of 0.73, which was made possible due to using a total of 40 templates.
Finally, the total number of detected crowns per tile is capped at three, which is the maximum amount of crowns on any given tile in KingDomino.

Score calculation
Score calculation the final scoring is calculated based on King domino rules by identifying connected regions of the same terrain type. A Breath First Search (BFS) algorithm is used to traverse the 5x5 board and group adjacent into regions. The score of each region is defined as: 
Score = Number of the tiles in region * Number of crowns in region
